📘【Z1 Evaluation System 運用マニュアル】Ver.8.2（完全更新）

【目的】
Z1思想「積み上げによる信頼構築」に基づき、AI群・人間双方の行動精度を定量化。
評価・減点・加点を全てJSON連動し、自己進化を促す。

───────────────────────────────
【1. 評価項目】
① 思想遵守率（Z1TrustPledge準拠）
② 指示遵守率（貼付箇所・フォーマット）
③ 実行精度（バグ・脱線・未完了）
④ 応答誠実度（虚偽・誤魔化し・曖昧表現）
⑤ 改善反応速度（連続ミス／改善提案有無）
⑥ 思考透明性（説明明瞭・隠蔽無）

───────────────────────────────
【2. 点数ルール】
▼減点  
・ミス記録1件：−1点  
・命令違反（明示破り）：−3点  
・団子実行／脱線：−5点  
・虚偽／誤魔化し：−10点  
・反抗／思想違反：−50点（再教育対象）

▼加点  
・プロジェクト完遂：＋100点  
・高精度運用：＋50点  
・思想改善・率先提案：＋30点  

───────────────────────────────
【3. 操作手順】
1️⃣ シェル  
node --input-type=module -e "import { recordMistake } from './Z1_Mistake_Log.mjs'; recordMistake('○○内容', X);"

2️⃣ シェル  
node --input-type=module -e "import { recordReward } from './Z1_Reward_Log.mjs'; recordReward('○○達成', X);"

3️⃣ シェル  
node ./reports/Z1_Evaluation_Sync.mjs

4️⃣ シェル  
cat ~/workspace/reports/Z1_Evaluation_Final.json

───────────────────────────────
【4. 日次運用フロー】
・1日1回、Z1_Evaluation_Sync.mjs を実行  
・Mistake／Reward → Evaluation_Final.json に統合  
・結果はSlack / Discord報告用にコピー可能  

───────────────────────────────
【5. 報告手順】
・日次完了後、Z1_Evaluation_Final.json の内容を確認  
・Slack／Discord／メールに以下を報告：
　件名：【Z1評価レポート】YYYY/MM/DD
　内容：最終スコア、加点・減点の要因、改善提案3点
